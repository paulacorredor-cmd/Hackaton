🏛️Reglas de Arquitectura

Arquitectura de 3 Capas

PresentaciónSolo renderizado UI y eventos. NUNCA lógica de negocio.
LógicaReglas de dominio, cálculos, validaciones de seguridad.
PersistenciaRepositorios oficiales. Siempre vía API, nunca conexión directa desde Front.

Seguridad

  • OAuth 2.0 + OIDC obligatorio
  • JWT firmados para comunicación entre capas
  • RBAC validado en Backend (no permisos hardcodeados)
  • Tokens en httpOnly cookies, nunca en localStorage
  • Datos sensibles enmascarados al 100% en respuestas API
  • Secretos en Variables de Entorno o Secret Manager
  • Comunicación solo por DNS, prohibido IPs fijas

Resiliencia y Desempeño

  • Frontend: respuesta < 3 segundos
  • Backend APIs: procesamiento < 15 segundos
  • Correlation-ID obligatorio en cada petición
  • Retries + Circuit Breakers para el 100% de errores
  • Rate Limiting para proteger componentes core
  • Versionamiento de APIs: /v1/api/, /v2/api/

🚫Patrones Prohibidos

  • Importar HttpClientModule en componentes (rompe interceptors)
  • Hardcodear colores/sombras/fuentes en vez de usar tokens --sb-ui-*
  • overflow: hidden en contenedores admin
  • Usar colores de Tailwind, Material o Bootstrap
  • Crear NgModules (solo standalone components)
  • Lógica de negocio en el Frontend
  • Conexión directa a BD desde el Front
  • API keys o credenciales en código fuente
  • Tokens en localStorage
